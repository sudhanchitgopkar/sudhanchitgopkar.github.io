calculate_purity(matches)¶

Calculates the purity of the cross-referenced matches based on the similarity score. The similarity score is defined as 1/(1+x) where x is the on-sky separation (in degrees) between the objects.

Parameters:

matches (pd.DataFrame) – The DataFrame containing matched rows with a ‘similarity_score’ column.

Returns:

An estimated purity score of the matches, represented as the average of the similarity scores.

Return type:

float

take_derivative(order: int) → DataFrame¶

Take integral derivative of given order on SDSS dataframe.

Takes derivatives of integer order of flux with respect to wavelength at each wavelength in dataframe, updates dataframe with added column containing derivatives, and returns updated dataframe.

Column names follow pattern “d*order*FdW*order*” (i.e. “d2FdW2”, “d3FdW3”) except when order is 1, then column name is “dFdW”.

Orders greater than 1 will also append columns for orders 1:order. (i.e. a call take_derivative(2) will create columns “d2FdW2” and “dFdW”.)

Derivatives are computed using an average finite difference formula:

Parameters:

order (int) – integral derivative order

Returns:

database with integral derivative column appended

Return type:

pd.DataFrame

Raises:

  • ValueError – if order is not an integer or is negative

  • TypeError – if order is not an integer

take_fractional_derivative(order: float) → DataFrame¶

Take fractional derivative of given order on SDSS dataframe.

Takes derivatives of fractional order of flux with respect to wavelength at each wavelength in dataframe, updates dataframe with added column containing derivatives, and returns updated dataframe.

Column names follow pattern “d{order}FdW{order}” (i.e. a call to take_fractional_derivative(1/2) will create column “d1/2FdW1/2”

This module uses Python differint package to compute pointwise Grünwald-Letnikov fractional derivatives.

Parameters:

order (float) – fractional derivative order

Returns:

database with fractional derivative column appended

Return type:

pd.DataFrame

Raises:

  • ValueError – if order is not a float or is negative

  • TypeError – if order is not a float

extract(param)¶

Extracts user provided metadata from the spec_obj params.

Given some user-defined parameter in spec_obj, this function will extract the value at that parameter and return it as a list. The list will consist of the spec_obj’s class as the first index, followed by the extracted value. An Exception will be raised if the db or spec_obj has not been initialized, while a KeyError will be raised if the spec_obj has no class attribute or does not contain the param requested. Finally, a TypeError will be raised if param is not an str.

Parameters:

param (str) – metadata to be extracted

Returns:

list <class, metadata requested>

Return type:

list

Raises:

  • Exception – if no database or spec_obj is found

  • TypeError – if param is not a string

  • KeyError – if param is not in spec_obj

load_spectrum(obj_id)¶

Load the spec_obj and spectrum dataframes from a given object id.

With a given object id, load the spec_obj, spectrum, and emission dataframes from the SDSS database. spec_obj contains metadata about the object, while spectrum contains the actual spectrum data. emission contains the emission/absorption line data. The spec_obj, spectrum, and emission dataframes are stored in the spec_obj, spectrum, and emission attributes respectively.

Parameters:

obj_id (int or str) – SDSS object id

Returns:

Raises:

  • ValueError – if obj_id is not found in the SDSS database

remove_outlier(columns=None, dropna=False, db='spectrum', exclude=None) → DataFrame¶

Remove outliers from specified columns in a DataFrame.

Identifies and replaces outliers with NaN in specified columns using the Z-score method. Outliers are values more than 3 standard deviations from the mean. Optionally, rows with NaNs post-removal can be dropped. Choose between db, spec_obj, spectrum, or continuum to remove outliers. Default is spectrum. Optionally, exclude certain columns from outlier removal. Performs outlier removal in place.

Parameters:

  • df (pd.DataFrame) – DataFrame from which to remove outliers.

  • columns (List[str]) – Columns to check for outliers. If empty, checks all columns.

  • dropna (bool) – If True, drops rows with NaNs after outlier removal. Default is False.

  • db (str) – Database to remove outliers from. Default is ‘spectrum’.

  • exclude (List[str]) – Columns to exclude from outlier removal.

Returns:

DataFrame with outliers removed.

Return type:

pd.DataFrame

Raises:

  • ValueError – if db is not a valid option, or does not exist

  • KeyError – if column is not in dataframe

add_graph_settings(graph_settings: dict) → None¶

Add graph settings for a visualizer.

Add a new dictionary of information about the graph to be made, e.g. labels, title, colors, markers. An example of a valid graph setting is:

available_settings = {

“x_label” : None, “y_label” : None, “title” : None, “spec_color” : colors, “cont_color” : colors, “spec_marker” : markers, “cont_marker” : markers, “spec_linestyle” : line_styles, “cont_linestyle” : line_styles

}

Available parameters include ‘x_label’, ‘y_label’, ‘title’, ‘spec_color’, ‘cont_color’, ‘spec_marker’, ‘cont_marker’, ‘spec_linestyle’, ‘cont_linestyle’.

Labels must adhere to matplotlib standards, as must colors and marker types.

Parameters:

graph_settings (dict) – A dictionary holding settings for the graph.

Raises:

  • AssertionError – if the graph settings are invalid.
